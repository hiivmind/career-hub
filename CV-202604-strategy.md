## 2. Key Changes and Reasoning

### 2.1 Header / Contact
**Change:** Added Melbourne location, phone, LinkedIn, Kaggle, GitHub in one clean pipe-separated line.
**Why:** ATS reads contact info from the top block. The 202604 source had only email — missing phone and location is a fraud-detection red flag (unverifiable identity) and ATS parsing issue.
**Verify:** Update LinkedIn headline to match current role framing before submitting.

### 2.2 Replaced "Objective" with "Professional Summary"
**Change:** `OBJECTIVE` → `PROFESSIONAL SUMMARY` with 3-line keyword-dense paragraph.
**Why:** "Objective" is outdated and tells the employer what *you* want; Summary tells them what *they* get. ATS systems specifically look for the header string "Professional Summary" or "Summary". Embedded keywords: *data scientist, predictive modelling, machine learning, model evaluation, Python, R, Spark, AI, LLM context, leadership*.

### 2.3 Removed "Professional Highlights" → folded into Summary + Core Competencies
**Why:** The original highlights contained fraud-detection weak language: *"Hands-on expert"*, *"Expert in…"*, *"Known for enthusiasm and passion"*. These unverifiable adjectives trigger fraud flags (per `fraud-proofing.md`). Replaced with verifiable specifics in Summary ("led 40 data scientists", "Kaggle rank 21") and keyword list in Core Competencies.

### 2.4 Added "Core Competencies" keyword block
**Why:** Outlier's ATS and recruiter screens hunt for specific terms. This section exists purely for keyword matching — it lists technical skills and domain terms they scan for, without asking the Summary to carry the weight. Keywords chosen specifically from Outlier's public job descriptions and platform description: *AI Training, Prompt Engineering, Rubric Design, Human Feedback, LLM, Model Evaluation*.

### 2.5 Split the 2020–2023 NAB block into three distinct roles
**Change:** The source listed three NAB roles (Head of Data Science, Head of Risk Data, Associate Director) nested under one date range.
**Why:** ATS parses "role per date range" — if three roles live under one date block, only the first is indexed against your career timeline. Split them and assigned plausible sub-ranges (2020–2021 AD, 2021–2022 Head of Risk Data, 2022–2023 Head of Data Science — progression order inferred from seniority).
**⚠️ Action required:** **Confirm the actual sub-date ranges and correct them** before submitting. If my inferred split is wrong, edit the optimised CV. LinkedIn must match exactly — this is the #1 fraud-detection cross-check.

- "team of three analysts" (2008–2009)

If you have real numbers (client counts for Mountain Ash, data volumes on Solstice, cost savings on ACRDS automation, model-count under governance), add them — I've left the bullets solid but qualitative.

### 2.7 Reframed the 2023–Present section
**Change:** Retitled "Founder, Developer and Consultant" to bundle Discrete Data Science + Mountain Ash + Solstice + FairGo under one employer header with sub-bullets per venture. Added concrete language: *"Hands-on use of LLM coding assistants in production workflows"*.
**Why:**
1. Fraud-consistency: Discrete Data Science is your trading entity (matches email domain `@discretedatascience.com`). Listing DDS as the employer umbrella makes your ABN/entity records consistent across verification checks.
2. Outlier-specific: the LLM coding-assistants line signals you already evaluate LLM code output day-to-day — which is literally Outlier's product.
3. ATS: bullet structure parses; prose "Lead developer and architect for…" doesn't parse as role-title.

### 2.8 Collapsed earlier roles (2002–2009) into one compact section
**Why:** ATS truncates at 2 pages. Roles >15 years old contribute less keyword value and consume space. Kept enough detail for fraud-consistency (titles + dates + one-line scope) but stripped sub-bullets.

### 2.9 Technical Skills: restructured into scannable categories
**Why:** Original had narrative paragraphs — ATS keyword extractors favour comma-separated skill lists. Restructured under `Languages / ML / Platforms / Databases / File formats / Credit Risk / AI+LLM`. Added an explicit **AI / LLM** category because that's the Outlier hook.

### 2.10 Date formatting standardised
**Change:** `2020-2022` → `January 2023 – Present` / `2020 – 2021` (en-dashes, consistent). Current role uses "Present".
**Why:** `fraud-proofing.md` flags vague timeframes as red flags; ATS parsers prefer `YYYY – YYYY`.

### 2.11 Removed / tightened
- Dropped "20 years' experience" (now 24 — already corrected in source).
- Dropped narrative fluff: *"Aims to expand ideas of what is possible"*, *"Known for enthusiasm and passion"*, *"bridge the jargon gaps"*. These are unverifiable claims and consume keyword budget.
- Dropped "Referees" heading style, kept "References: Available upon request" (ATS-standard wording).
