# ATS Optimization: Technical Formatting & Keyword Strategy

## File Format & Storage

**Requirement**: `.docx` (Word) format preferred unless job explicitly specifies PDF.

**Why**: ATS systems parse Word documents more accurately than PDFs. PDFs lose formatting data during conversion, making content invisible to parsing systems.

**Action**:
- Save as `.docx` before submission
- Do NOT submit as `.pdf` unless explicitly required
- Never use Google Docs or cloud-native formats

## Layout Structure: ATS-Parsing Requirements

### Mandatory Rules

- **Single-column layout** (no multi-column designs)
- **Left-aligned text** throughout
- **Standard margins**: 0.5–1 inch
- **Font size**: 10.5–12 point
- **No tables, text boxes, or columns**

### Prohibited Elements

- Graphics, icons, or decorative elements
- Colored headers or text boxes
- Columns or side-by-side content
- Special characters (→, •◦, ✓, etc.)
- Headers/footers with page numbers or location info in the document
- Abbreviations without spelling them out at first use (except widely known ones like "IBM")

### Font Choices

**Preferred**: Arial, Calibri, Helvetica, Times New Roman, Garamond

**Avoid**: Unusual fonts, decorative typefaces, or anything without universal system support

**Why**: ATS reads resumes left-to-right, top-to-bottom. Graphics and tables break this parsing flow, causing the system to skip content or misinterpret structure.

## Section Structure & Naming

Use these exact standard section headers. ATS recognizes these common names:

```
[Your Name]
[Phone] | [Email] | [LinkedIn URL] | [City, State]

PROFESSIONAL SUMMARY
[2-3 lines of high-impact summary with keywords]

CORE COMPETENCIES
[Listed as keywords, not narrative]

PROFESSIONAL EXPERIENCE
[Reverse chronological order]

EDUCATION
[Degree, Institution, Graduation Date]

CERTIFICATIONS / LICENSES
[If applicable]
```

**Critical rule**: Do NOT use creative section names like "About Me," "What I've Done," or "Professional Journey." The ATS needs standard headers.

## Keyword Extraction & Natural Integration

### The Task

Extract 15-25 high-value keywords from the job description, then integrate them naturally throughout the CV.

### Process

1. **Scan the job description** for repeated terms and phrases
2. **Identify hard skills** (specific tools, software, technologies, methodologies)
3. **Identify soft skills** (leadership, communication, problem-solving, cross-functional collaboration)
4. **Identify domain jargon** and industry-specific terminology
5. **Extract job titles or role designations** mentioned in the posting

### Integration Strategy

- Place keywords in **Professional Summary** (first 2-3 lines, high density but natural)
- Distribute keywords across **3-5 achievement bullets** in Professional Experience
- Create **Core Competencies section** that explicitly lists discovered keywords as skills

### Critical Balance

The goal is to match keywords *naturally* without "keyword stuffing." Recruiters and modern NLP-based ATS systems detect unnatural keyword density and treat it as a red flag.

## Contact Information Header Format

```
[Your Full Name - Large, Bold]
[City, State] | [Phone Number] | [Email] | [LinkedIn URL] | [GitHub/Portfolio URL if relevant]
```

**Format rules**:
- Name is the largest text on the page (14–16pt)
- No personal pronouns (e.g., "John's CV")
- Do NOT include: Photo, age, birthdate, marital status, national origin, religion
- LinkedIn URL should be clean: `linkedin.com/in/yourname` (not the full messy URL)

**Why**: ATS parses contact info cleanly; personal details can trigger compliance issues. Clean URL format matters because recruiters click these links.

## Length & Page Limits

**Ideal**: 1–2 pages

**Risk**: ATS systems sometimes truncate at page 2. If you have 3+ pages, consolidate.

**Proofread**: Spelling errors are automatic disqualifications in many ATS systems.
